新闻

Redisson 3.10.1 和 2.15.1 发布

两个版本的更新内容相同，除了修复一些 bug 之外，还包含以下新特性：

  • cachedKeySet, cachedValues, cachedEntrySet and getCachedMap 方法已经添加到 RLocalCachedMap 对象。
  • 支持 Hibernate 5.4 。
  • 通过 field 来查找 LiveObjects。
  • 允许检查锁是否被线程持有

再看这条新闻 Redisson 成为 GitHub 里星星最多的 Redis Java 客户端，群众的眼光是雪亮的。

Redis在秒杀功能的实践

Redis 在秒杀中的两处典型应用，一是缓存购买者热身数据、抢购者的查询信息等。另一处是负责资源库存剩余数量、秒杀资源抢占等核心业务。

Redis 用作缓存时，要谨慎选择 LRU 策略。这时关闭 RDB 和 AOF，也是追求极限性能的可用操作。

采用 Redis 队列来实现秒杀时，lpush/llen 等操作的复杂度都是 O(1)。

Redis实现分布式锁的正确姿势

为了实现分布式锁，需要确保锁同时满足：

  • 互斥性。在任意时刻，只有一个客户端能持有锁。
  • 不会死锁。即使一个客户端没有主动释放锁，也需要保证后续其它客户端能够加锁成功。
  • 加锁和解锁必须是同一个客户端，客户端不能释放把别人加的锁。

为此，需要做的是：

  • 给 value 值为一个唯一标示，保证加锁和解锁的为同一对象。
  • 锁需要设置过期时间，避免死锁。
  • setnx 和设置过期时间之间，要保证原子性。

携程Redis海外机房数据同步实践

携程开源的 XPipe 是一套 Redis 多机房系统。简单来说，这是伪 slave 的方案，即实现 Redis 协议，伪装成为 slave，让 master 推送数据至伪 slave。

这篇文章在 XPipe 的基础上，解决了公网传输的问题。整体架构中用到了 TCP 的反向代理，并尝试了不同的拥塞控制算法，最终圈定了 BBR。
