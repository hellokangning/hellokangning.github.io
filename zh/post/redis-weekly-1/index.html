Redis主体流程分析

从源码层面分析了 Redis Server 的启动过程以及请求的处理和响应。Redis 服务器的状态保存在 redisServer 这个结构体中，其中的成员变量 commands 字典保存了 Redis 的命令表。

Redis 处理客户端请求是在函数 acceptTcpHandler() 完成的。这个函数通过 anetTcpAccept() 接收客户端请求，然后调用 acceptCommandHandler() 来处理客户端请求。

为什么不搞集群服务也能实现Redis高可用？

对哨兵的各种使用姿势做了总结。引入哨兵之后，客户端首先连接的是哨兵，在获取到 master 信息之后，会建立与 Redis Server 的连接。所以，哨兵也要避免单点问题，哨兵节点个数应该大于等于3，且最好为奇数。另外，尽可能在不同的物理机上部署哨兵节点。
